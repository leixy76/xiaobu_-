⭐️ Claude Code 的 35 个实用技巧 (持续更新)

交互式翻转卡片版 | 融合全球重度用户实战经验，彻底改变你的开发范式。

一、核心交互心法

1

治疗师倾诉式提问

治疗师倾诉式提问

心法：像对治疗师一样，毫无保留地倾诉你的整个问题。开启一个新会话时，不要吝啬文字，把问题的背景、你的尝试、你的目标、你的顾虑，一次性全部告诉它。

“我目前的 Claude Code 之舞是这样开始的：开启一个全新的会话，像和治疗师交谈一样，把我所有的问题都倾倒出来。”

这种“信息 dump”的方式能让 AI 在一开始就获得最全面的上下文，从而制定出更精准、更宏观的执行计划。

2

用语音输入突破带宽

用语音输入突破带宽

观点：如果你还在用键盘打字，你几乎肯定没有提供足够的上下文。语音输入是带宽的倍增器，它能让你毫不费力地将更多的想法、顾虑和细节传递给 AI。

3

把需求说具体

把需求说具体

避免使用“修复这个漏洞”这类笼统的指令。提供具体的上下文和问题描述能让 AI 更精准地定位问题。例如，说“修复用户登录时不输入密码出现的空指针错误”。

4

复杂需求分步执行

复杂需求分步执行

对于大型或复杂的需求，建议将其拆解成多个小步骤，分步下发指令。这便于审查和测试，也能避免超出 AI 的上下文限制。

5

先理解再动手

先理解再动手

在修改或添加功能前，先让 Claude 理解你的项目。让 AI 先分析，再动手，可以得到更精准、高效的辅助。例如让它分析数据库结构或错误处理机制。

二、上下文管理

6

积极清空上下文

积极清空上下文

这是最重要的技巧之一。 当你要开始一个新任务或话题时，务必使用 /clear 命令。

高级工作流：当对话即将达到压缩阈值时，命令 Claude “将当前要点记录在 notes.md 中”，然后手动开启一个新会话。这比等待压缩更快、更干净。

7

使用便笺和暂存盘

使用便笺和暂存盘

利用文件系统作为 AI 的“短期记忆”或“草稿纸”。

  • 便笺 (Scribble Notes): 让它随时“把...讨论要点，记录在 `branch-analysis.md` 文件里”。
  • 暂存盘 (Scratchpad): 命令 Claude 将所有文件修改、增删等实时记录到一个 scratchpad.md 文件中，方便随时恢复工作记忆。
8

精简记忆文件

精简记忆文件 (CLAUDE.md)

请谨慎使用 /init 命令自动生成，因为它会包含大量无关信息。最佳实践：手动维护这个文件，只在里面放置最高级别、最重要的指令，如“请始终使用 TypeScript”。

9

手动压缩上下文

手动压缩上下文 (/compact)

当对话历史变得很长时，使用 /compact 命令可以清除历史记录但保留上下文摘要。这是一个在不清空所有内容的情况下，减少 Token 使用量的温和替代方案。

10

恢复历史会话

恢复历史会话 (/resume)

如果会话意外中断，不用担心。可以使用 /resume 命令或启动时使用 claude -r 来恢复之前的对话，选择要继续的上下文。

三、工作流与效率

11

Git 即检查点

Git 即检查点 (Checkpointing)

Claude Code 缺乏内置的“检查点”功能。最佳解决方案是使用 Git 作为你的手动检查点系统。在让 AI 开始一项重要任务前，先执行一次 git commit。如果结果不满意，你可以轻松地回退到上一个“存档点”。

12

CC in Cursor 工作流

CC in Cursor 工作流

在 Cursor 编辑器内部的终端中运行 Claude Code，这是一个强强联合的工作流。

核心优势：你可以充分利用 Cursor 极其便利的 Diff 可视化审查和合并冲突解决工具，来审查 Claude Code 生成的每一处修改，提供了极大的安全感和控制力。

13

使用免授权模式

使用免授权模式

启动时带上 --dangerously-skip-permissions 参数，开启 Bypassing Permissions 模式，让 Claude 自动执行所有操作，无需手动确认。

14

善用快捷键

善用快捷键

  • /: 查看所有斜杠命令。
  • 上下方向键: 查看命令历史。
  • Tab: 命令快速补全。
  • Shift + ?: 查看所有快捷键列表，发现隐藏功能！
15

自定义快捷命令

自定义快捷命令

为常用或复杂的任务创建自定义命令，是提升效率的关键。例如创建 /project:build（项目级） 或 /user:gsave（用户级） 等。

16

文件暂存解决复制难题

文件暂存解决复制难题

Claude Code 终端的复制粘贴体验不佳。一个简单的解决方法是：让 Claude Code 先将需要的内容写入到一个临时文件（如 `temp.txt`），然后你再从那个文件中复制。

17

随时打断与引用

随时打断与引用

  • 打断任务: 按下 ESC 键，即可立即停止 Claude 的当前工作。
  • “分叉”对话: 连按两次 ESC 键，可以调出历史消息，并从任一处重新开始或编辑。
18

始终保持更新

始终保持更新

Anthropic 团队对 Claude Code 的更新频率非常高。请务必定期运行升级命令：

sudo npm install -g @anthropic-ai/claude-code

四、高级策略与玩法

19

超越编码：万物皆可编辑

超越编码：万物皆可编辑

思维转变：Claude Code 不仅仅是“代码”工具，它是一个强大的、能与你的本地文件系统交互的“通用编辑”工具。 你可以用它来帮你写周报、整理笔记、生成营销文案等。

20

Opus 规划, Sonnet 执行

Opus 规划, Sonnet 执行

这是一个兼具质量和效率的黄金工作流。

  1. 使用 /model opus 切换到 Opus。
  2. 按下 Shift+Tab 进入计划模式，让 Opus 制定高质量计划。
  3. 切换回 /model sonnet，让更快的 Sonnet 完成执行工作。

这样既利用了 Opus 的强大规划能力，又享受了 Sonnet 的高速度和低成本。

21

模型选择策略

模型选择策略

  • Sonnet 4: 90% 的情况下都足够好，尤其擅长处理长上下文、Python 和前端代码，速度更快。
  • Opus 4: 当 Sonnet 陷入困境或遇到非常棘手的 bug 时，切换到 Opus 解决核心难题。
22

规划先行：制定PRD

规划先行：制定PRD

在开始任何大型项目之前，花时间与 AI 一起制定一份产品需求文档（PRD）。关注“什么”而非“怎么做”，这会极大提升最终产出的质量。

23

激活深度思考模式

激活深度思考模式

通过使用 /think, /think hard, /ultrathink 等命令，可以分配更多思考预算给 AI，以应对复杂问题。

24

使用计划与自动编辑模式

使用计划与自动编辑模式

通过按下 Shift + Tab 键，可以在普通模式、自动接受编辑模式和计划模式之间循环切换，灵活应对不同任务阶段的需求。

25

利用子代理并行处理

利用子代理并行处理

在处理大型代码库的搜索等任务时，你可以指示 Claude 使用子代理（sub-agents）并行执行。它会通过内部的“Task tool”部署多个轻量级代理同时工作，大幅提升效率。

26

善用 MCP 扩展能力

善用 MCP 扩展能力

MCP 协议允许 Claude Code 与外部工具交互。例如添加 context7 MCP 来获取最新的库文档，或使用 Playwright server 自动化前端开发并形成“编码-截图-审查”的闭环。

五、实用功能与命令

27

对话式 Git/Linux 交互

对话式 Git/Linux 交互

用自然语言执行 Git 和 Linux 命令。例如，使用默认的 /review 命令审查分支的 diff，或者直接说“请检查一下和 main 分支的差异”。

28

发送图片处理

发送图片处理

Claude Code 支持处理图片。你可以直接将图片粘贴到命令行中（Mac 下快捷键是 Ctrl + V），并附上提示词，例如分析一张错误截图的原因。

29

使用 @ 提及文件

使用 @ 提及文件

这是一个基础但极易忽略的功能。在提问时，使用 @ 符号可以快速地将一个或多个文件加入到上下文中，比拖拽或复制路径更高效。

30

进入 Bash/无头模式

进入 Bash/无头模式

  • Bash 模式: 在命令前加上 !，直接执行 shell 命令，不消耗 Token。
  • 无头模式: 使用 claude -p "你的问题" 直接在终端运行一次性命令并退出，适合脚本化操作。
31

权限管理命令

权限管理命令

除了启动时的参数，你还可以在会话中使用 /permissions 命令来动态调整当前会话的权限策略。

32

模型切换

模型切换 (/model)

使用 /model 命令可以在不同模型间（如 Sonnet 和 Opus）灵活切换。

33

查看消耗情况

查看消耗情况 (/cost)

使用 /cost 查看当前会话的消耗，或使用 ccusage 工具进行全局统计。

34

回滚代码

回滚代码

发送“回滚”指令可以撤销上一步的文件更改。但更推荐使用 Git 作为更可靠的检查点机制。

35

巧用 Hooks 提升体验

巧用 Hooks 提升体验

在配置文件中设置 Hooks，在 AI 回复完成时执行脚本，例如为不同项目设置不同的完成提示音，提升多任务处理时的感知力。

AI 不会淘汰程序员，但不会用 AI 的程序员除外。掌握这些思维和技巧，你将拥有未来。
